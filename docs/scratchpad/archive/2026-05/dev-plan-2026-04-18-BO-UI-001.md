## API du composant (figée)

```ts
export type ResponsiveBreakpoint = 'sm' | 'md' | 'lg' | 'xl' | '2xl';
export type ResponsiveAlign = 'left' | 'right' | 'center';
export type ResponsiveDensity = 'compact' | 'normal';

export interface ResponsiveColumn<T> {
  id: string;
  header: string;
  width?: number; // px fixe
  minWidth?: number; // px minimum
  hideBelow?: ResponsiveBreakpoint;
  align?: ResponsiveAlign;
  cell: (row: T) => React.ReactNode;
  tooltipWhenHidden?: (row: T) => string;
}

export interface ResponsiveDataTableProps<T> {
  columns: ResponsiveColumn<T>[];
  data: T[];
  rowKey: (row: T) => string;
  actions?: (row: T) => React.ReactNode;
  actionsWidth?: number; // défaut 90
  emptyState?: React.ReactNode;
  loading?: boolean;
  skeletonRows?: number; // défaut 3
  onRowClick?: (row: T) => void;
  density?: ResponsiveDensity; // défaut 'compact'
  className?: string;
}
```

## Mapping Tailwind pour `hideBelow`

Tailwind n'a pas de classe native "hidden below lg". On utilise le pattern
`hidden <bp>:table-cell` :

| hideBelow | Header className        | Cell className          |
| --------- | ----------------------- | ----------------------- |
| `sm`      | `hidden sm:table-cell`  | `hidden sm:table-cell`  |
| `md`      | `hidden md:table-cell`  | `hidden md:table-cell`  |
| `lg`      | `hidden lg:table-cell`  | `hidden lg:table-cell`  |
| `xl`      | `hidden xl:table-cell`  | `hidden xl:table-cell`  |
| `2xl`     | `hidden 2xl:table-cell` | `hidden 2xl:table-cell` |

Tailwind JIT génère ces classes statiquement si elles apparaissent dans le
code. On utilise un Record pour garantir l'extraction :

```ts
const HIDE_BELOW_HEADER: Record<ResponsiveBreakpoint, string> = {
  sm: 'hidden sm:table-cell',
  md: 'hidden md:table-cell',
  lg: 'hidden lg:table-cell',
  xl: 'hidden xl:table-cell',
  '2xl': 'hidden 2xl:table-cell',
};
```

## Sticky actions colonne

Pattern standard CSS :

- `position: sticky; right: 0;` sur `th` + `td` actions
- `z-index: 10` sur header row, `z-index: 1` sur cells
- `background-color: hsl(var(--background))` pour éviter la transparence
- Box-shadow gauche : `shadow-[-4px_0_8px_-4px_rgba(0,0,0,0.1)]` pour
  indiquer le scroll horizontal disponible

Le `<Table>` shadcn a déjà `overflow-auto` sur son wrapper, donc le sticky
fonctionne out-of-the-box.

## Density

- `compact` (défaut) : `text-xs` sur cells, `py-2` sur row (override de
  `py-2.5` shadcn)
- `normal` : `text-sm` + `py-2.5` (hérite shadcn)

Implémenté via un `className` appliqué conditionnellement sur chaque
`<TableCell>`.

## Loading state

Si `loading=true` : rendre `skeletonRows` lignes de `<Skeleton className="h-10 w-full" />` à la place du body. Les colonnes restent visibles (header intact).

## Empty state

Si `!loading && data.length === 0` : si `emptyState` fourni, le rendre
centré dans une seule TableCell `colSpan={columns.length + (actions ? 1 : 0)}`.
Sinon fallback texte `Aucune donnée`.
