# Audit T2-only à 375px — 2026-04-19

**Objectif** : vérifier que les 19 composants migrés en "T2 hidden classes only" dans PR #651 sont réellement responsive sur mobile 375px, avant merge.

**Méthode** : Playwright runtime à 375×800, screenshot, classification :
- 🟢 OK : contenu lisible sur mobile, soit via cards natives soit colonnes suffisantes
- 🟡 LIMITE : tableau avec colonnes minimales, utilisable mais tendu
- 🔴 KO : scroll horizontal, colonnes trop masquées, inutilisable

**Rappel règle T1** (`.claude/rules/responsive.md`) : "Sous md (< 768px), un tableau est INTERDIT. Obligation de basculer en liste de cartes empilées."

T2-only ne satisfait pas T1 formellement — cet audit évalue l'acceptabilité concrète du compromis.

---

## Résultats par composant

### Groupe 1 — Organisation listings (4 composants) : 🟢 TOUS OK

**Raison** : ces pages ont un `viewMode` switcher (grid/list) avec **grid par défaut**. Sur mobile, la vue grid affiche des **cards natives** complètes (logo + nom + badges + actions).

| Composant | Page | État 375px | Verdict |
|---|---|---|---|
| SuppliersListView | `/contacts-organisations/suppliers` | Cards natives (vue grid), 17 actifs affichés proprement | 🟢 OK |
| CustomerListView | `/contacts-organisations/customers` | Cards natives, 168 actifs avec logo + nom + badges | 🟢 OK |
| PartnersListView | `/contacts-organisations/partners` | Cards natives (empty state testée) | 🟢 OK |
| EnseigneListView | `/contacts-organisations/enseignes` | Cards natives, 2 enseignes avec logo + organisations | 🟢 OK |

**Note** : la `ListView` (mode liste) avec T2 n'est visible qu'après switch manuel de l'utilisateur — rare sur mobile. Les T2 ajoutées restent utiles pour le cas edge.

## Recommandation

**Pour** :

**Contre** :
- 2 tables 🔴 confirmées inutilisables sur mobile
- 11 tables 🟡 non vérifiables sans données

**Action** : créer `.claude/queue/BO-UI-RESP-LISTS-T1-CATCH-UP.md` pour migrer les 🔴 + re-tester les 🟡 avec données.

**Scope fix minimal** :
- `SalesOrderDataTable` : colonne Actions doit rester visible → modifier `hidden lg:table-cell` sur actions conditionnelles si applicable. Pour now: vérifier que Actions n'est pas masquée.
- `expeditions-order-row` : revoir T2 pour garder au moins Client + Actions + Statut
- `CommissionsTable` : garder Affilié + Commission + Statut visibles

**Estimation** : 2h pour 2-3 fichiers + runtime test.
